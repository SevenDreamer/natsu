# Plan 06-02: Script Library

**Phase:** 6 - Automation Core
**Requirement:** AUTO-02 - 用户可以从脚本库保存和运行脚本

## Objective

实现脚本库管理功能，支持脚本的创建、保存、编辑和执行，复用 Phase 5 的命令安全检查机制。

## Scope

### In Scope
- 脚本元数据数据库表
- 脚本文件存储管理
- 脚本执行（复用 ExecuteCommandTool 安全检查）
- 脚本标签分类
- 参数化脚本支持

### Out of Scope
- 定时执行 (Phase 7)
- 文件变更触发 (PLAN-03)

## Tasks

### Task 1: 数据库 Schema 扩展

**File:** `natsu/src-tauri/src/db/schema.rs`

```sql
CREATE TABLE IF NOT EXISTS scripts (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    description TEXT,
    script_path TEXT NOT NULL,
    interpreter TEXT DEFAULT 'bash',
    tags TEXT,
    parameters TEXT,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_scripts_name ON scripts(name);
```

### Task 2: 脚本模型

**File:** `natsu/src-tauri/src/models/script.rs` (新建)

```rust
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Script {
    pub id: String,
    pub name: String,
    pub description: Option<String>,
    pub script_path: String,
    pub interpreter: String,
    pub tags: Vec<String>,
    pub parameters: Vec<ScriptParameter>,
    pub created_at: i64,
    pub updated_at: i64,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ScriptParameter {
    pub name: String,
    pub description: Option<String>,
    pub default_value: Option<String>,
    pub required: bool,
}

#[derive(Debug, Clone, Deserialize)]
pub struct ScriptExecutionInput {
    pub script_id: String,
    pub parameters: std::collections::HashMap<String, String>,
    pub timeout: Option<u64>,
}
```

### Task 3: 后端脚本 API

**File:** `natsu/src-tauri/src/commands/script.rs` (新建)

```rust
#[tauri::command]
pub fn list_scripts(conn: State<Connection>) -> Result<Vec<Script>, String>

#[tauri::command]
pub fn get_script(id: String, conn: State<Connection>) -> Result<Script, String>

#[tauri::command]
pub fn create_script(
    name: String,
    description: Option<String>,
    content: String,
    interpreter: String,
    tags: Vec<String>,
    parameters: Vec<ScriptParameter>,
    storage_path: State<String>,
    conn: State<Connection>,
) -> Result<Script, String>

#[tauri::command]
pub fn update_script(
    id: String,
    name: Option<String>,
    description: Option<String>,
    content: Option<String>,
    tags: Option<Vec<String>>,
    conn: State<Connection>,
) -> Result<Script, String>

#[tauri::command]
pub fn delete_script(
    id: String,
    storage_path: State<String>,
    conn: State<Connection>,
) -> Result<(), String>

#[tauri::command]
pub async fn execute_script(
    input: ScriptExecutionInput,
    storage_path: State<String>,
    conn: State<Connection>,
) -> Result<ScriptExecutionResult, String>
```

执行脚本时：
1. 读取脚本内容
2. 替换参数变量 `{{param_name}}`
3. 使用 `ExecuteCommandTool::get_safety_info()` 检查安全性
4. 返回安全信息供前端确认（如需要）
5. 执行并返回结果

### Task 4: 脚本存储

脚本文件存储在 `<storage_path>/scripts/` 目录下：
- 文件名使用 UUID，避免冲突
- 支持多种解释器：bash, sh, python, node 等
- 检测文件类型自动选择解释器
